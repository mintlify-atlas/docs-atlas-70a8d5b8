title: Lab
description: TypeScript types for interactive lab environments
---

The `ParsedLab` type represents a fully resolved lab configuration with services, workspace settings, and file scaffolding.

## ParsedLab

<ParamField path="title" type="string" required>
  Lab title
</ParamField>

<ParamField path="instructions" type="string" required>
  Markdown instructions rendered in the lab sidebar
</ParamField>

<ParamField path="filesPath" type="string" required>
  Absolute path to scaffold files directory
</ParamField>

<ParamField path="solutionPath" type="string | undefined" required>
  Absolute path to solution directory, or undefined if no solution exists
</ParamField>

<ParamField path="workspace" type="'fresh' | 'continue'" required>
  Workspace lifecycle mode. `'fresh'` resets to scaffold on each provision. `'continue'` preserves previous state.
</ParamField>

<ParamField path="testCommand" type="string" required>
  Shell command to run tests (e.g., `'npm test'`)
</ParamField>

<ParamField path="openFiles" type="readonly string[]" required>
  List of file paths to open in editor on lab start
</ParamField>

<ParamField path="services" type="readonly ResolvedService[]" required>
  Docker/Podman services to provision
</ParamField>

<ParamField path="setup" type="readonly string[]" required>
  Shell commands to run after workspace provision
</ParamField>

## ResolvedService

A fully resolved container service (preset or custom).

<ParamField path="name" type="string" required>
  Service name (used as container name)
</ParamField>

<ParamField path="image" type="string" required>
  Docker image (e.g., `'postgres:17-alpine'`)
</ParamField>

<ParamField path="port" type="number" required>
  Container internal port
</ParamField>

<ParamField path="hostPort" type="number" required>
  Host port to expose
</ParamField>

<ParamField path="env" type="Readonly<Record<string, string>>" required>
  Environment variables for the container
</ParamField>

<ParamField path="healthcheck" type="string" required>
  Healthcheck command (e.g., `'pg_isready -U postgres'`)
</ParamField>
